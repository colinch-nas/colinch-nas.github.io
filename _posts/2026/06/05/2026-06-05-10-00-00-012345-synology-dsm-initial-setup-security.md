# Synology NAS 처음 설정 — DSM 7.2 초기 보안 설정 완전 정복

![Synology NAS 서버 보안 설정](https://images.unsplash.com/photo-1558494949-ef010cbdcc31?w=800&q=80)

NAS 박스 개봉하고 케이블 꽂으면 5분 안에 DSM 접속까지 된다. 근데 그 상태로 외부 접속 열었다가 랜섬웨어 맞은 사람이 한둘이 아니다. 초기 보안 설정은 귀찮아도 반드시 해야 한다.

## 환경 정보

| 항목 | 버전/모델 |
|------|----------|
| NAS 모델 | Synology DS923+ |
| DSM 버전 | DSM 7.2.2-72806 Update 2 |
| 작업 시점 | 2026년 6월 |

## 기본 포트는 바꿔라

DSM 기본 접속 포트는 HTTP 5000, HTTPS 5001이다. 전 세계 봇들이 이 포트를 24시간 스캔하고 있다. 처음 NAS 설치하고 얼마 안 됐는데 로그 보면 수십 개 접속 시도가 쌓여 있다.

**제어판 → 로그인 포털 → DSM** 으로 이동해서 포트를 바꾼다. 8001/8002 같은 번호 말고 좀 더 독특한 번호로 설정하는 게 낫다. 49000번대처럼 잘 안 쓰는 범위를 권장한다.

```bash
# 변경 후 접속 주소 예시
http://192.168.1.100:49000    # HTTP
https://192.168.1.100:49001   # HTTPS
```

포트 바꾸면 잠시 접속이 끊긴다. 당황하지 말고 새 포트로 다시 접속하면 된다.

## admin 계정은 비활성화한다

DSM 설치할 때 admin 계정이 기본으로 생성된다. 이 계정명은 공개 정보나 마찬가지라서 브루트포스 공격 1순위다. 새 관리자 계정을 만들고 admin은 꺼버려야 한다.

1. **제어판 → 사용자 및 그룹 → 사용자** 에서 새 계정 생성
2. 새 계정을 **administrators** 그룹에 추가
3. 새 계정으로 로그인한 뒤 admin 계정 비활성화

처음엔 admin 비활성화하면 뭔가 망가질 것 같아서 그냥 뒀는데, 실제로는 아무 문제 없다. 오히려 그냥 두는 게 더 위험하다.

## 2단계 인증 필수

**제어판 → 보안 → 계정** 탭에서 **2단계 인증 강제 적용**을 켠다. Google Authenticator나 Authy 앱으로 TOTP를 등록하면 된다.

![Synology DSM 2단계 인증 설정 화면](https://images.unsplash.com/photo-1558618666-fcd25c85cd64?w=800&q=80)

근데 여기서 함정이 있다. 가족 공용 NAS라면 모든 계정에 2FA를 강제하기 전에 가족들한테 먼저 알려야 한다. 모르고 강제 적용했다가 아내가 갑자기 접속 못 한다고 연락 온 적 있다.

## 방화벽 설정

**제어판 → 보안 → 방화벽** 에서 외부 접속을 허용할 포트만 명시적으로 열고, 나머지는 차단하는 방식으로 설정한다.

필요한 포트 예시:
- DSM 관리 포트 (변경한 것)
- HTTPS 443 (리버스 프록시 사용 시)
- SSH 22 (사용한다면, 기본 포트는 바꿀 것)

```bash
# SSH 포트 변경 위치
제어판 → 터미널 및 SNMP → SSH 서비스 활성화 → 포트 변경
```

SSH는 쓸 일 없으면 그냥 꺼두는 게 낫다. 방화벽 설정할 때 **모든 거부** 규칙을 맨 아래에 추가하는 걸 빠뜨리기 쉬운데, 이게 없으면 방화벽이 실질적으로 동작하지 않는다.

## 자동 차단 설정

**제어판 → 보안 → 보호** 탭에서 **자동 차단**을 켠다. 로그인 실패 횟수 제한(예: 5분 내 5회 실패)을 설정하면 브루트포스 공격이 자동으로 차단된다.

차단된 IP 목록은 같은 화면에서 확인하고 해제할 수 있다. 처음 설정할 때 너무 빡빡하게 잡으면 나 자신이 차단당한다. 5분 내 5회 정도가 무난하다.

## DSM 업데이트 자동화

**제어판 → 업데이트 및 복구** 에서 **중요 업데이트 자동 설치**를 켠다. DSM 보안 패치는 빠르게 적용할수록 좋다.

단, 메이저 버전 업그레이드(예: DSM 7.x → 7.y)는 자동으로 하지 말고 릴리즈 노트 확인 후 수동으로 하는 편이 안전하다. 대규모 업데이트 직후에는 패키지 호환성 문제가 생기는 경우가 있다.

## 삽질했던 부분

포트 변경하고 공유기 포트 포워딩도 바꿔줘야 하는데 이걸 빠뜨렸다. NAS에서 포트 바꿨는데 외부 접속이 안 된다고 한 시간 삽질하고 알아낸 건데, 결국 공유기 설정을 안 바꾼 거였다.

방화벽에서 국가 차단도 해볼 수 있다. Synology에서 한국 외 IP를 모두 차단하는 지역 기반 방화벽 규칙을 지원한다. 실제로 적용하면 로그의 공격 시도가 눈에 띄게 줄어든다.

## 한 줄 정리

기본 포트 변경, admin 비활성화, 2FA, 방화벽, 자동 차단 — 이 다섯 가지만 해도 일반적인 스크립트 공격은 대부분 막힌다.
