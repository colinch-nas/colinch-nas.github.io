QNAP TS-464를 쓰는데 [3-2-1 백업 글]({% post_url 2026-06-21-10-00-00-209865-qnap-nas-backup-321-rule-implementation %})을 쓸 때까지만 해도 QTS 5.1.8이었다가, 그 사이 자동 업데이트로 5.2.7까지 올라가 있었다. 그런데 6월 17일에 QNAP이 QSA-26-10이라는 보안 권고를 내놨는데, 거기 적힌 취약점 중 하나가 인증 없이도 공격 가능하다고 해서 바로 업데이트했다.

## 환경 정보

| 항목 | 값 |
|------|----------|
| NAS 모델 | QNAP TS-464 |
| 업데이트 전 QTS | 5.2.7 |
| 업데이트 후 QTS | 5.2.9.3499 |
| 영향받는 제품 | QTS, QuTS hero, QuTS cloud, QVP(QVR Pro) |

## 왜 이게 필요한가

[QNAP 공식 보안 권고 QSA-26-10](https://www.qnap.com/en/security-advisory/qsa-26-10)에 QTS, QuTS hero, QuTS cloud, QVP 전반에 걸친 취약점 14건이 올라왔다. 이 중 두 개가 특히 신경 쓰였다.

**CVE-2025-59382** — 비밀번호 재설정 링크를 외부에서 조작할 수 있는 URL 인젝션 취약점이다. 공격자가 재설정 링크를 가로채거나 변조해서 계정을 탈취할 수 있다.

**CVE-2026-26241** — 청크 업로드(chunked file upload) 시 파일명을 비정상적으로 길게 넣으면 스택 버퍼 오버플로가 발생한다. 인증 없이도 트리거 가능하다는 게 문제다. CGI 프로세스가 죽는 수준이지만, 이런 종류의 메모리 취약점은 나중에 RCE로 발전하는 경우가 많아서 가볍게 볼 게 아니다.

영향받는 버전은 QTS 5.2.7, QuTS hero h5.2.8, QuTS cloud c5.2.8, QVP 2.7.1이고, 패치된 버전은 QTS 5.2.9.3499, QuTS hero h5.2.9, QuTS cloud C5.2.9, QVP 2.8.0이다. TS-464는 QTS 5.2.7이라 정확히 영향받는 버전에 걸려 있었다.

## 업데이트 방법

QTS 제어판에서 직접 확인하고 업데이트했다.

```
제어판 → 시스템 → 펌웨어 업데이트
→ "지금 업데이트 확인" 클릭
→ QTS 5.2.9.3499 표시되면 다운로드 후 적용
```

자동으로 안 잡히면 QNAP 다운로드 센터에서 모델명으로 직접 검색해서 받아도 된다.

```
https://www.qnap.com/ko-kr/download
→ 모델 검색: TS-464
→ 펌웨어 탭에서 QTS 5.2.9.3499 다운로드
→ 제어판 → 펌웨어 업데이트 → 수동 업데이트 → 다운로드한 파일 업로드
```

업데이트 적용 전에 시스템 설정 백업은 기본이다.

```
제어판 → 시스템 → 백업/복원 → 설정 백업
```

업데이트는 재부팅을 동반하니 진행 중인 Container Station 작업이나 백업 스케줄이 겹치지 않는 시간에 했다. 나는 새벽 시간대로 Task Scheduler에 등록해서 처리했다.

## 업데이트만으로 끝내지 않은 것들

패치 적용은 기본이고, 추가로 점검한 부분이 있다.

**비밀번호 재설정 경로 점검**: CVE-2025-59382가 재설정 링크 조작이라서, myQNAPcloud 계정과 QNAP ID에 걸린 이메일이 실제 본인 계정인지, 2단계 인증이 켜져 있는지 다시 확인했다. 제어판 → 사용자 → 2단계 인증에서 OTP 앱 연동이 가능하다.

**외부 노출 면 줄이기**: Security Counselor 앱으로 다시 스캔했더니 "포트 포워딩 위험" 항목이 하나 떠 있었다. 예전에 테스트하다가 켜놓고 잊어버린 포트였다. 외부 접속은 포트 포워딩보다 Tailscale 같은 VPN을 쓰는 게 공격 표면을 줄이는 데 훨씬 낫다. ([Tailscale로 홈서버 외부 접속하는 법]({% post_url 2026-06-18-10-00-00-172830-tailscale-vpn-home-server-remote-access %}) 참고)

**자동 업데이트 옵션 재확인**: 제어판 → 시스템 → 펌웨어 업데이트에서 "중요 보안 업데이트 자동 설치"를 켰다. 메이저 버전 업그레이드는 자동 적용 안 되고, 보안 패치 수준만 자동으로 들어오게 설정할 수 있다.

![QNAP 보안 점검 대시보드](https://images.unsplash.com/photo-1544197150-880d4212a0c4?w=800&q=80)

## 삽질했던 부분

**업데이트 후 Container Station 재시작 필요**: 업데이트 적용 후 재부팅까지 했는데도 Docker 컨테이너 몇 개가 자동으로 안 올라왔다. Container Station 앱을 수동으로 한 번 열었다가 닫으니까 그제야 컨테이너들이 정상 기동했다. QTS 업데이트할 때마다 반복되는 증상이라 이제는 그냥 업데이트 체크리스트에 넣어놨다.

**수동 다운로드 시 모델명 확인 안 하면 헛수고**: 다운로드 센터에서 비슷한 모델명(TS-464 vs TS-464C 등) 펌웨어를 잘못 받으면 업데이트 화면에서 "호환되지 않음" 에러만 뜨고 왜 안 되는지 알려주지 않는다. 모델명 끝자리까지 정확히 확인해야 한다.

## 한 줄 정리

QSA-26-10은 인증 없이 트리거 가능한 메모리 취약점이 포함돼 있어서 업데이트를 미룰 이유가 없다. QTS 5.2.9.3499로 올리고, 겸사겸사 2단계 인증과 포트 포워딩 설정도 같이 점검하는 게 낫다.
