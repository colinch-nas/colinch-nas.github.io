# Synology NAS 처음 설정 — DSM 7.2 기준 초기 설정과 보안 완전 정리

![Synology NAS 설정 화면](https://images.unsplash.com/photo-1558618666-fcd25c85cd64?w=800&q=80)

NAS를 처음 켜면 설정이 생각보다 많다. DSM 설치하고 관리자 계정 만들고 끝이 아니다. 기본 설정 그대로 인터넷에 연결된 NAS는 외부 공격에 열려 있는 거나 마찬가지다. 실제로 Synology NAS를 대상으로 한 무차별 대입 공격은 꾸준히 발생한다. 이 글에서는 처음 개봉부터 기본 보안 설정까지 DSM 7.2.2 기준으로 정리한다.

## 환경 정보

| 항목 | 내용 |
|------|------|
| NAS 모델 | Synology DS923+ (다른 모델도 동일) |
| DSM 버전 | DSM 7.2.2 |
| 설정 접근 | 브라우저 http://[NAS IP]:5000 |

## 처음 부팅 후 DSM 설치

1. NAS를 LAN에 연결하고 전원을 켠다
2. 같은 네트워크에 연결된 PC에서 `find.synology.com` 접속하거나 Synology Assistant 앱 실행
3. 장치가 검색되면 클릭 후 "설치" 진행
4. DSM 7.x 최신 버전이 자동으로 다운로드·설치됨 (10~15분 소요)
5. 관리자 계정 이름과 비밀번호 설정

여기서 관리자 계정 이름을 `admin`으로 하면 안 된다. 공격자들이 가장 먼저 시도하는 이름이다. `nas_admin`, `homeserver` 등 추측하기 어려운 이름을 쓴다.

## 해야 할 기본 보안 설정 7가지

### 1. 기본 admin 계정 비활성화

DSM에는 `admin`이라는 내장 계정이 있다. 이게 활성화된 상태에서는 공격 대상이 된다.

`제어판 → 사용자 및 그룹 → admin → 편집 → 계정 비활성화`

### 2. 2단계 인증 활성화

관리자 계정 로그인에 OTP를 추가한다.

`우측 상단 프로필 → 개인 → 계정 → 2단계 인증 활성화`

Google Authenticator, Authy 등 TOTP 앱과 연동한다. 한 번 설정하면 로그인할 때마다 앱에서 코드를 확인해야 한다.

### 3. 자동 차단 설정

로그인 실패가 반복되면 해당 IP를 자동으로 차단한다.

`제어판 → 보안 → 자동 차단`

```
활성화: 켜기
로그인 횟수: 5회
시간 내: 5분
차단 기간: 영구 차단 권장
```

설정 후 "허용 목록"에 본인 집 IP를 추가해두면 실수로 잠기는 걸 방지할 수 있다.

### 4. 방화벽 활성화

외부에서 접근 가능한 포트를 제한한다.

`제어판 → 보안 → 방화벽 → 방화벽 규칙 편집`

허용할 것만 남기고 나머지는 기본적으로 거부. 처음엔 아래 정도만:

| 포트 | 용도 | 허용 여부 |
|------|------|----------|
| 5000 | DSM HTTP | 내부만 |
| 5001 | DSM HTTPS | 필요 시만 |
| 22 | SSH | 필요 시만 |
| 445 | SMB | 내부만 |

외부 접근이 필요한 경우엔 VPN(Tailscale 등)을 통해 접속하는 방식이 직접 포트 개방보다 안전하다.

### 5. HTTPS 강제 사용

`제어판 → 네트워크 → DSM 설정 → HTTP 연결을 HTTPS로 자동 리디렉션`

Let's Encrypt 인증서는 `제어판 → 보안 → 인증서`에서 도메인이 있으면 발급 가능하다.

### 6. 알림 설정

로그인 시도, 보안 경고를 이메일로 받을 수 있다.

`제어판 → 알림 → 이메일`

Gmail 기준: SMTP 서버 `smtp.gmail.com`, 포트 587, TLS 사용. Gmail에서 앱 비밀번호를 발급받아 입력해야 한다.

### 7. DSM 자동 업데이트 설정

보안 패치는 빠르게 적용하는 게 중요하다.

`제어판 → 업데이트 및 복원 → DSM 업데이트 → 자동으로 중요 업데이트만 설치`

## 초기 설정 완료 체크리스트

```
[ ] admin 계정 비활성화 또는 새 이름으로 변경
[ ] 2단계 인증 활성화
[ ] 자동 차단 설정 (5회 실패 시 영구 차단)
[ ] 방화벽 활성화
[ ] HTTPS 강제 리디렉션
[ ] 알림 이메일 설정
[ ] 자동 업데이트 설정
[ ] Snapshot 또는 백업 스케줄 설정
```

마지막 백업 설정은 빠뜨리기 쉬운데, 이게 없으면 랜섬웨어나 디스크 오류 시 복구 방법이 없다.

## 삽질했던 부분

자동 차단 설정하고 허용 목록에 집 IP를 안 넣었다가 본인이 잠긴 적이 있다. 모바일 데이터로 접속해서 차단 해제했다. 집 IP가 유동 IP인 경우 주기적으로 바뀌니까, 허용 목록에 IP 대신 IP 범위(예: 192.168.1.0/24)를 등록하면 내부 네트워크에서는 잠기지 않는다.

## 한 줄 정리

NAS 개봉하면 admin 계정 비활성화, 2단계 인증, 자동 차단 이 세 가지를 제일 먼저 한다.
