이 그림에서 봐야 할 건 DSM 업데이트 전에 디스크 모델과 스토리지 풀 상태를 확인해야 한다는 점이다.

Synology DSM 7.3으로 올릴 생각이라면 업데이트 버튼보다 `저장소 관리자 -> HDD/SSD` 화면을 우선 열어야 한다. 2026년 7월 8일 기준 Synology 공식 발표는 DSM 7.3에서 2025년형 DiskStation Plus, Value, J 시리즈가 검증되지 않은 서드파티 드라이브로 설치와 스토리지 풀 생성을 지원한다고 설명한다. 다만 M.2 기반 스토리지 풀과 캐시는 여전히 HCL(호환성 목록)에 있는 드라이브가 필요하다고 못 박았다.

내가 헷갈렸던 건 "서드파티 HDD 가능"이 "아무 SSD나 전부 가능"이라는 뜻은 아니라는 점이다. DS925+ 같은 2025년형 모델에서 3.5인치 HDD를 꽂는 상황과, M.2 NVMe로 스토리지 풀을 만들려는 상황은 조건이 다르다.

## 업데이트 전 확인표

| 확인 항목 | DSM 7.3에서 봐야 할 부분 | 바로 진행해도 되는 기준 |
|---|---|---|
| 모델 연식 | 제어판 -> 정보 센터 | 2025년형 DiskStation Plus/Value/J인지 확인 |
| HDD/SSD 상태 | 저장소 관리자 -> HDD/SSD | 불량 섹터, 경고, 재연결 이력 없음 |
| 스토리지 풀 | 저장소 관리자 -> 스토리지 | 재동기화, 확장, 스크럽 작업 없음 |
| M.2 사용 | 저장소 관리자 -> SSD 캐시/풀 | HCL 등재 모델인지 확인 |
| 백업 | Hyper Backup 또는 USB 복사 | 최근 24시간 안에 완료된 백업 있음 |

업데이트는 아래 순서로 하는 편이 덜 불안하다. 특히 외부 접속, Drive, Photos, Docker 패키지를 같이 쓰는 NAS라면 재부팅 한 번으로 끝난다고 가정하면 안 된다.

```text
1. 저장소 관리자에서 디스크 상태 캡처
2. Hyper Backup 작업 수동 실행
3. 패키지 센터에서 중요 패키지 업데이트 확인
4. DSM 7.3 릴리스 노트 확인
5. DSM 업데이트 적용
6. 재부팅 후 스토리지 풀, 공유 폴더, 패키지 로그 확인
```

공식 릴리스 노트는 staged rollout(단계적 배포)로 나올 수 있다. 내 NAS 화면에 아직 안 보인다면 다운로드 센터에서 수동 파일을 받아 올릴 수도 있지만, 운영 NAS에서는 굳이 첫날 수동 업로드를 고집할 이유가 적다. 나는 업무 파일이 있는 장비는 최소 하루 두고, 커뮤니티에서 같은 모델 이슈가 없는지 확인한 뒤 올린다.

## 디스크 교체 계획이 있으면 더 조심한다

새 HDD를 사서 DSM 7.3 업데이트와 동시에 스토리지 풀을 바꾸는 건 피하는 게 낫다. 문제가 생겼을 때 원인이 DSM인지, 디스크인지, 기존 풀 상태인지 갈라내기 어렵다.

```text
좋은 순서: DSM 업데이트 -> 하루 관찰 -> 새 디스크 추가
나쁜 순서: 새 디스크 장착 -> DSM 업데이트 -> 풀 확장 동시 진행
```

2025년형 Synology NAS를 새로 샀다면 디스크를 꽂기 전에 Synology 호환성 목록에서 모델명과 용량을 확인한다. 예를 들어 같은 IronWolf라도 용량, 펌웨어, SATA/M.2 구분에 따라 표시가 달라질 수 있다. "인식은 된다"와 "장기 운영에 적합하다"는 다른 말이다.

주의할 점은 세 가지다. 첫째, DSM 7.3 설치 후 이전 버전으로 쉽게 되돌릴 수 있다고 생각하면 안 된다. 둘째, M.2 스토리지 풀은 3.5인치 HDD 정책과 따로 본다. 셋째, 외부 접속을 열어 둔 NAS는 업데이트 직후 DDNS, 인증서, 방화벽 규칙까지 확인해야 한다.

짧게 정리하면 이렇다. DSM 7.3은 2025년형 DiskStation의 서드파티 3.5인치 HDD 운용 폭을 넓혔지만, M.2 제한까지 사라진 건 아니다. 업데이트 전에는 디스크 상태, 백업, HCL, 진행 중인 스토리지 작업을 확인한다. 디스크 교체와 DSM 업그레이드는 같은 날 묶지 않는 게 덜 고생한다.
