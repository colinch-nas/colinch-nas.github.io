TrueNAS 26-BETA.2는 운영 NAS에 바로 올릴 버전이 아니다. 2026년 6월 17일 공개된 공식 버전 노트에도 Early Release(테스트와 피드백용 초기 릴리스)는 중요한 작업에 쓰지 말라고 못 박혀 있다. 그래도 SMB 성능 개선, 네트워크 FEC 설정, ACL 프리셋 변경은 홈서버 사용자에게 바로 영향을 주는 항목이라 테스트 VM에서 확인할 가치는 있다.

내 기준에서는 실데이터 풀을 가져오지 않고 Proxmox VE 8.4 위에 빈 디스크 2개짜리 VM을 만들었다. CPU 2코어, RAM 8GB, 부트 디스크 32GB, 테스트 데이터 디스크 2개면 충분했다. NIC는 VirtIO로 두고, 실제 NAS와 같은 VLAN에 붙였다. 여기서 중요한 건 성능 벤치마크가 아니라 메뉴 위치와 SMB 권한 동작을 미리 보는 것이다.

## 테스트 VM 구성

TrueNAS ISO로 설치한 뒤 웹 UI에 접속하면 Storage에서 새 풀을 만든다. 이름은 운영 NAS와 다르게 `tank-test`처럼 붙인다. 실수로 복제 스크립트를 돌렸을 때 운영 경로와 섞이지 않게 하기 위해서다.

SMB 확인용 데이터셋은 이렇게 나눴다.

```text
tank-test/share-family
tank-test/share-media
tank-test/share-backup
```

사용자는 `nasuser`, 그룹은 `family` 하나만 만들었다. ACL은 기본 프리셋을 그대로 저장하지 말고 화면에 자동으로 추가되는 로컬 사용자·관리자 그룹을 확인한다. 26-BETA.2에서는 내장 ACL 프리셋이 로컬 및 디렉터리 서비스 그룹을 더 적극적으로 포함한다. 이 부분을 안 보고 저장하면 의도보다 넓은 권한이 들어갈 수 있다.

## SMB 공유 확인 순서

Shares에서 Windows SMB Share를 추가하고 `share-family`만 공개한다. 서비스 자동 시작을 켠 뒤 윈도우와 macOS에서 각각 접속한다.

```bash
smbutil statshares -a
```

macOS에서는 Finder로 사진 폴더처럼 파일이 많은 디렉터리를 열어본다. 공식 변경점에 따르면 macOS SMB 디렉터리 목록 처리에서 불필요한 시스템 호출이 줄었다. 체감 확인은 단순하다. 같은 폴더를 25.10 VM과 26-BETA.2 VM에 각각 복사해두고 Finder 첫 목록 표시 시간을 비교하면 된다. 숫자를 예쁘게 만들려고 캐시가 남은 두 번째 열기 결과를 쓰면 의미가 없다.

## 네트워크 설정에서 볼 것

26-BETA.2는 지원 NIC에서 FEC(Forward Error Correction, 고속 링크 오류 보정) 모드를 UI에서 설정할 수 있게 했다. 10GbE 이상을 쓰는 홈랩이면 Network > Interfaces에서 항목이 보이는지 확인한다. 다만 저가 2.5GbE 어댑터나 VirtIO NIC에서는 안 보일 수 있다. 안 보이는 게 버그라고 단정하면 여기서 삽질한다.

운영 NAS로 올리기 전 체크는 세 가지만 통과하면 된다.

- SMB 접속 계정이 의도한 데이터셋만 볼 수 있는가
- macOS Finder에서 대량 파일 폴더가 멈추지 않는가
- 네트워크 인터페이스 설정을 저장해도 관리 UI 접속이 끊기지 않는가

## 주의할 점

업그레이드 테스트라고 해서 운영 풀을 가져오면 안 된다. ZFS 풀을 새 버전에서 열고 기능 플래그를 올리면 예전 버전으로 되돌리는 길이 막힐 수 있다. 앱과 컨테이너도 마찬가지다. 이번 테스트의 목적은 TrueNAS 26-BETA.2의 UI와 SMB 동작을 보는 것이지, 운영 환경을 미리 이사하는 게 아니다.

공식 문서 기준 날짜는 2026년 6월 17일이고, 확인한 버전은 TrueNAS 26-BETA.2다. 지금은 테스트 VM에서 SMB와 ACL만 확인하고, 운영 NAS는 안정 릴리스가 나온 뒤 설정 백업을 만들고 올리는 쪽이 맞다.
