UGREEN NASync DXP4800 GT처럼 10GbE 포트가 두 개 달린 NAS를 샀다면 파일 복사부터 하지 말고 네트워크 병목부터 잡아야 한다. 2026년 7월 2일 기준 UGREEN 공식 스펙은 AMD Ryzen Embedded R2514, 4베이 SATA, 2개 NVMe 슬롯, 듀얼 10GbE를 내세운다. 최근 리뷰도 10GbE에서 1GB/s 안팎 전송을 이야기하지만, 집 네트워크에서는 스위치와 PC 랜카드 설정이 더 자주 발목을 잡는다.

## 왜 속도가 안 나오나

10GbE NAS를 10GbE 스위치에 꽂아도 윈도우 탐색기 복사가 280MB/s에서 멈추는 일이 있다. 내 경우 원인은 NAS가 아니라 PC 쪽 2.5GbE 포트였다. 또 Cat.5e 케이블은 짧으면 버티기도 하지만, 벽면 포트와 패치 패널을 지나면 링크가 5Gbps로 떨어지는 경우가 있었다.

처음에는 아래 순서로 확인하면 된다.

```bash
# PC 또는 미니PC에서 링크 속도 확인
ethtool enp3s0 | grep Speed

# macOS라면 네트워크 서비스명 확인 후
networksetup -getMedia "USB 10G Ethernet"
```

## 고정 IP와 MTU부터 맞춘다

NAS 관리 화면에서 10GbE 포트에 고정 IP를 준다. 예를 들어 기존 공유기망이 `192.168.0.0/24`라면 10GbE 전용망은 `192.168.10.0/24`처럼 분리한다. 인터넷 게이트웨이는 비워도 된다. NAS와 작업 PC를 같은 10GbE 스위치에 물리고 아래처럼 잡는다.

```text
NAS 10GbE: 192.168.10.10 / 255.255.255.0
PC 10GbE:  192.168.10.20 / 255.255.255.0
Gateway:  비움
DNS:      비움
```

Jumbo Frame(MTU 9000)은 모든 장비가 지원할 때만 켠다. NAS, 스위치, PC 중 하나라도 1500이면 오히려 끊김이 생긴다. 켠 뒤에는 큰 패킷으로 확인한다.

```bash
ping 192.168.10.10 -M do -s 8972
```

실패하면 MTU 1500으로 되돌린다. 10GbE는 MTU 1500에서도 충분히 빠르다.

## iperf3로 디스크를 빼고 잰다

파일 복사는 디스크, 캐시, SMB 설정이 섞인다. Docker에서 iperf3 서버를 띄워 네트워크만 따로 본다. DXP4800 GT는 Docker 활용을 전제로 사는 사람이 많으니 이 테스트가 가장 깔끔하다.

```bash
docker run --rm -p 5201:5201 networkstatic/iperf3 -s
```

PC에서는 이렇게 잰다.

```bash
iperf3 -c 192.168.10.10 -P 4
```

결과가 8~9Gbps 근처면 네트워크는 정상이다. 2.3Gbps 근처면 2.5GbE 링크에 붙은 것이다. 900Mbps 근처면 케이블, 스위치 포트, 공유기 경유 여부부터 본다.

## SMB는 멀티채널보다 단일 경로로 본다

SMB Multichannel(여러 네트워크 경로를 동시에 쓰는 기능)은 좋지만, 처음 설정에서 바로 켜면 원인 분리가 어렵다. 10GbE 단일 IP로 공유 폴더를 연결하고 큰 파일 하나를 복사한다.

```text
\\192.168.10.10\video
```

HDD RAID5라면 500~800MB/s도 자연스럽다. NVMe 캐시나 SSD 볼륨이면 1GB/s에 더 가까워진다. 여기서 속도가 낮으면 NAS 앱 설치보다 디스크 풀, 스냅샷 작업, 백그라운드 인덱싱부터 확인한다.

주의할 점은 UGREEN UGOS Pro가 Synology DSM만큼 오래 검증된 운영체제는 아니라는 점이다. Docker, SMB, 백업은 가능해도 업무 원본을 바로 옮기기보다 기존 NAS에서 한 번 더 백업을 유지한다. 정리하면 10GbE NAS 첫날에는 앱 설치보다 링크 속도, MTU, iperf3, SMB 단일 경로 순서로 확인하는 게 시간을 아낀다.
