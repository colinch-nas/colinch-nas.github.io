![TrueNAS 홈서버 설치와 관리](https://images.unsplash.com/photo-1558494949-ef010cbdcc31?w=1200&q=80)

TrueNAS 25.10.4는 이제 모니터와 키보드를 NAS에 연결하지 않아도 TrueNAS Connect의 Web Install로 설치할 수 있다. 2026년 7월 업데이트에서 TrueNAS는 25.10.4를 새 장비 설치에 권장 버전으로 안내했고, 테스트상 설치 자체는 5분 안에 끝난다고 밝혔다. 다만 USB 부팅과 같은 네트워크에 있는 조건은 여전히 필요하고, 설치 후 SSL 인증서를 직접 바꿔야 하는 지점도 있다.

![TrueNAS Connect Web Install 흐름](https://images.unsplash.com/photo-1518770660439-4636190af475?w=1200&q=80)
이 그림에서 볼 부분은 설치 작업이 클라우드에서 실행되는 것이 아니라, 브라우저와 집 안의 설치 대상 장비 사이에서 진행된다는 점이다.

## 준비한 환경

이번 글의 기준은 미니 PC에 TrueNAS SCALE 25.10.4를 새로 설치하는 경우다.

| 항목 | 기준 |
|---|---|
| 설치 버전 | TrueNAS SCALE 25.10.4 |
| 설치 미디어 | TrueNAS ISO를 기록한 USB |
| 네트워크 | 설치 PC와 NAS가 같은 공유기·VLAN |
| 대상 디스크 | 운영체제 전용 SSD 1개 |
| 계정 | TrueNAS Connect 계정 + `truenas_admin` |

TrueNAS Connect의 Web Install은 25.10 이상을 대상으로 한다. NAS가 USB를 자동으로 부팅하지 않는 보드라면 이 방법은 발견 단계에서 멈춘다. BIOS에서 USB Boot Priority를 올려두고, 기존 데이터가 있는 디스크는 설치 대상에서 분리하는 편이 안전하다.

## USB 부팅 후 Web Install 실행

TrueNAS ISO를 USB에 기록한 뒤 NAS에 꽂고 전원을 켠다. 별도 화면이 없어도 되지만, 첫 부팅이 끝날 때까지 2~3분 정도 기다려야 한다. 그다음 PC에서 TrueNAS Connect에 로그인하고 `Install TrueNAS` → `Install Assistant`를 연다.

`Detect System`을 누르면 시스템 이름, 설치할 TrueNAS 버전, 모델을 입력하는 창이 나온다. 같은 네트워크 세그먼트에 있고 mDNS(로컬 네트워크에서 장비 이름을 자동 발견하는 기능)가 동작하면 `System found`가 표시된다.

설치 대상 디스크를 크기로 확인한 뒤 선택한다. 이 단계에서 디스크의 모든 내용이 지워진다. 데이터 디스크가 여러 개 꽂혀 있다면 모델명과 용량을 메모해 두고, 가능하면 설치 SSD만 연결한 상태로 진행하는 게 낫다. 같은 크기의 부팅 디스크가 두 개면 설치 미러 옵션도 보이지만, 홈서버 첫 구축이라면 데이터 풀과 부팅 디스크를 혼동하지 않는 것이 우선이다.

## 관리자 계정과 재부팅

설치 마법사에서 `truenas_admin` 비밀번호를 만들고 호스트 이름과 위치 태그를 입력한다. 비밀번호는 Connect 계정과 다르게 만든다. 설치가 끝나면 USB를 제거했다는 체크박스를 선택하고 재부팅한다. 장비가 다시 올라오면 Connect 대시보드에서 `Connecting` 상태를 거쳐 시스템이 나타난다.

여기서 내가 처음 헷갈렸던 부분은 Connect가 보인다고 해서 TrueNAS UI의 HTTPS 설정까지 끝난 것은 아니라는 점이다. 현재 Web Install 문서에도 설치 후 기본 `truenas_default` 인증서를 다른 인증서로 바꾸라는 안내가 있다.

```text
TrueNAS UI 접속 → System Settings → Certificates
→ GUI SSL Certificate → 기본 인증서가 아닌 생성된 인증서 선택
→ 저장 후 TrueNAS Connect 대시보드 새로고침
```

인증서 변경 뒤 `Open TrueNAS UI`가 정상적으로 열리는지 확인한다. 화면이 열리지 않으면 NAS의 내부 IP로 직접 접속해 인증서 상태를 확인한다. 외부 접속을 위해 443 포트를 바로 포워딩할 필요는 없다. 초기 설치와 관리가 목적이면 Tailscale 같은 VPN을 먼저 구성하고, 포트포워딩은 꼭 필요한 서비스에만 적용한다.

## 설치 후 바로 할 점검

- `System Settings → General`에서 시간대와 NTP를 `Asia/Seoul` 기준으로 확인한다.
- `Credentials → Local Users`에서 관리자 계정의 2단계 인증을 설정한다.
- 부팅 SSD와 데이터 풀을 구분하고, 데이터 풀 생성 전 디스크 시리얼을 다시 확인한다.
- `System Settings → General → Manage Configuration`에서 초기 설정 파일을 내려받는다.
- SMART 테스트와 주기적인 스크럽 일정을 설정한다.

TrueNAS Connect 7월 업데이트의 핵심은 여러 장비를 한 화면에서 관리하는 기능보다, 화면 없는 미니 PC에도 설치 진입 장벽을 낮춘 Web Install이다. 다만 로컬 네트워크와 USB 부팅은 필수이고, SSL 인증서가 자동으로 완성되지 않을 수 있다. 설치가 빠르다는 이유로 디스크 확인과 설정 백업을 생략하면 안 된다.
